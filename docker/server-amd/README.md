# 📦 Media Server Stack (qBittorrent, Samba, File Manager)

## 🚀 Overview
This project sets up a **self-hosted media server** using **Docker Compose**. It includes:
- **qBittorrent with VPN**: Secure torrent downloads via WireGuard VPN.
- **Samba File Sharing**: Simple network file sharing for seamless access.
- **File Manager**: Web-based interface for managing files.

## ✨ Features
- ✅ **Encrypted torrenting** with VPN protection.
- ✅ **Network file sharing** via Samba (SMB protocol).
- ✅ **Web-based file management** with Filebrowser.
- ✅ **Persistent storage** for media files.

## 🏗 Architecture
```
+----------------------+
|  qBittorrent-VPN    |  --> Secured torrenting via WireGuard with Web UI (8080)
|      (8080)         |
+----------------------+
        |
        v
+----------------------+

+----------------------+
        |
        v
+----------------------+
|   File Manager      |  --> Web UI for managing files (port 8081)
| (Filebrowser)       |
+----------------------+
```

## ⚙ Installation & Configuration

### **1️⃣ Prerequisites**
- **Docker & Docker Compose** installed on your system.
- **A VPN provider** that supports WireGuard and provides configuration files.

### **2️⃣ Setup**
#### **Step 1: Create `.env` File**
Before deploying, create a `.env` file and populate it with the necessary environment variables:

```ini
PUID=1000
PGID=1000
TZ=America/New_York
VPN_ENABLED=yes
VPN_PROV=custom
VPN_CLIENT=wireguard
LAN_NETWORK=192.168.1.0/24

SAMBA_USER=plex
SAMBA_PASS=yourpassword
```

#### **Step 2: Deploy the Stack**
Run the following command to start the services for the first time:

```sh
docker-compose up -d
```

#### **Step 3: Configure WireGuard**
- Wait for the **qBittorrent-VPN container** to initialize.
- Once the folder `/docker/qbittorrent-vpn/wireguard/` is created, **stop the stack**:

  ```sh
  docker-compose down
  ```

- Place your **WireGuard configuration file** inside `/docker/qbittorrent-vpn/wireguard/` and rename it to **`wg0.conf`**.

- Restart the stack:

  ```sh
  docker-compose up -d
  ```

## 🔑 Credentials

### **qBittorrent Web UI**
- **URL**: `http://<server-ip>:8080`
- **Username**: `admin`
- **Password**: (Check logs using Docker)
  ```sh
  docker logs qbittorrent-vpn | grep -i password
  ```
  Alternatively, look inside `/docker/qbittorrent-vpn/config/` for configuration files.

### **Samba File Sharing**
- **Windows**: `\\<server-ip>\SharedMedia`
- **macOS/Linux**: `smb://<server-ip>/SharedMedia`
- **Login Credentials**:
  - **Username**: `plex`
  - **Password**: Set in `.env` file

### **File Manager**
- **URL**: `http://<server-ip>:8081`
- **Username**: `admin`
- **Password**: `admin`

## 🔒 Security Considerations
- **Built-in Kill Switch**: The `binhex/arch-qbittorrentvpn` container includes a kill switch to **prevent leaks if the VPN disconnects**.
- **Check Your External IP**:  
  Verify your IP address with:
  ```
  http://checkmyip.torrentprivacy.com/
  ```
- **Firewall Considerations**:  
  - Limit **exposed ports** to trusted devices.
  - Secure **Samba access** with proper permissions.
